# 百物語アプリケーション 仕様書

## 1. 概要

### 1.1 コンセプト
日本の伝統的な怪談イベント「百物語」をデジタル化し、参加者が順番に怪談を語り、ろうそくを消して（または灯して）いくインタラクティブなアプリケーション。

### 1.2 技術スタック
| 項目 | 技術 |
|------|------|
| フロントエンド | React（TypeScript） |
| UIライブラリ | タッチ操作最適化（後述） |
| バックエンド | FastAPI (Python) |
| データベース | PostgreSQL |
| 文字起こし | Whisper（ローカルモデル） |
| デプロイ | Snowflake SPCS |
| コンテナ | Docker |

### 1.3 対象デバイス
- **主要**: iPad（タッチパネル操作）
- **副次**: PC/Macブラウザ

---

## 2. 機能仕様

### 2.1 イベント管理

#### イベント作成
| 項目 | 設定内容 |
|------|----------|
| イベント名 | 自由入力 |
| 開催日 | 日付選択 |
| ろうそく本数 | 1〜100（自由設定） |
| 持ち時間 | 分単位で設定（例: 2分） |
| モード | Get Dark（消灯）/ Get Bright（点灯） |
| パスワード | 任意設定（イベント単位） |
| 背景画像 | アップロードまたはデフォルト選択 |
| ろうそく画像 | 点灯/消灯の2枚セット選択または登録 |
| イベントアイコン | アップロードまたはデフォルト |

#### 参加者管理
| 項目 | 内容 |
|------|------|
| ニックネーム | 自由入力 |
| アイコン | 選択またはアップロード |
| 追加タイミング | イベント中いつでも追加可能 |

#### イベントアクセス
- 一意のURL発行（例: `/event/{event_id}`）
- パスワード設定時は入力画面を表示
- 個人認証なし（イベント単位のみ）

### 2.2 イベント進行画面

#### ろうそく配置
```
┌─────────────────────────────────────┐
│  🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯  │
│  🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯  │
│  🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯  │
│  ...                              │
│  🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯 🕯  │
└─────────────────────────────────────┘
```
- 本数に応じて正方形に近い配置を自動計算
- 例: 100本 → 10×10、50本 → 7×8、25本 → 5×5
- 左上から右下へ順番に番号付け
- タップで枠を開く

#### ろうそくの状態
| モード | 未実施 | 実施済み |
|--------|--------|----------|
| Get Dark | 点灯🔥 | 消灯💨 |
| Get Bright | 消灯 | 点灯🔥 |

### 2.3 トーク画面（枠タップ時）

#### 画面遷移
```
[枠タップ] → [話者選択画面] → [トーク画面] → [完了] → [メイン画面]
```

#### 話者選択画面
```
┌─────────────────────────────────────┐
│        第○話 - 話者を選択          │
├─────────────────────────────────────┤
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐   │
│  │👤│ │👤│ │👤│ │👤│ │👤│   │
│  │Aさん││Bさん││Cさん││Dさん││Eさん│   │
│  └───┘ └───┘ └───┘ └───┘ └───┘   │
│                                     │
│  ┌───┐ ┌───┐ ┌───┐                │
│  │👤│ │👤│ │ + │ ← 参加者追加    │
│  │Fさん││Gさん│└───┘                │
│  └───┘ └───┘                       │
└─────────────────────────────────────┘
```
- 参加者アイコンをグリッド表示
- タップで話者を選択
- 「+」ボタンで新規参加者追加

#### トーク画面
```
┌─────────────────────────────────────┐
│  第○話                    [02:00]  │ ← タイマー（カウントダウン）
├─────────────────────────────────────┤
│                                     │
│         🔴 録音中...                │
│                                     │
│  ┌─────────────────────────────┐   │
│  │                             │   │
│  │   リアルタイム文字起こし    │   │
│  │   （表示領域）              │   │
│  │                             │   │
│  └─────────────────────────────┘   │
│                                     │
│  [🔴 強制録音開始]  [✓ 完了]        │
└─────────────────────────────────────┘
```

#### 録音ロジック
```
枠を開く
  ↓
自動録音開始
  ↓
┌─ 10秒未満で閉じた場合 → 録音破棄、再度開くと新規録音
│
└─ 10秒以上経過 → 録音確定、以降は上書き不可
                   （閉じても録音データ保持）
  ↓
完了ボタンで録音停止 → Whisperで文字起こし
```

#### タイマー
- 設定時間からカウントダウン表示
- 0になったら画面上で通知（色変化やアニメーション）
- 強制終了はしない

### 2.4 達成画面

#### トリガー
- 全ろうそくの状態が変化したとき（全消灯 or 全点灯）

#### 演出
- 軽いアニメーション（CSS/GIF）
- 特別な達成画像の表示
- モードに応じた演出
  - Get Dark: 暗転 → 何かが現れる演出
  - Get Bright: 全点灯 → 明るく祝福演出

---

### 2.5 過去イベント閲覧

#### 一覧画面
- 開催日順でイベント一覧表示
- イベントアイコン、名前、開催日、ろうそく本数

#### 詳細画面
- ろうそく配置を再現（全て完了状態）
- 各枠タップで話者と文字起こしを表示
- 文字起こしの編集機能あり
  - 編集ボタン押下で編集モードに
  - 保存/キャンセルで確定
  - **上書き録音ボタンは表示しない**

### 3.2 テーブル定義

#### events
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | VARCHAR(255) | イベント名 |
| event_date | DATE | 開催日 |
| candle_count | INTEGER | ろうそく本数 |
| time_limit_seconds | INTEGER | 持ち時間（秒） |
| mode | VARCHAR(20) | 'dark' or 'bright' |
| password_hash | VARCHAR(255) | パスワードハッシュ（NULL可） |
| background_image_url | TEXT | 背景画像URL |
| candle_on_image_url | TEXT | 点灯ろうそく画像URL |
| candle_off_image_url | TEXT | 消灯ろうそく画像URL |
| event_icon_url | TEXT | イベントアイコンURL |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### participants
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| event_id | UUID | イベントID（FK） |
| nickname | VARCHAR(100) | ニックネーム |
| icon_url | TEXT | アイコンURL |
| created_at | TIMESTAMP | 作成日時 |

#### talks
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| event_id | UUID | イベントID（FK） |
| slot_number | INTEGER | 枠番号（1〜candle_count） |
| speaker_id | UUID | 話者ID（FK、NULL可） |
| transcript | TEXT | 文字起こしテキスト |
| is_completed | BOOLEAN | 完了フラグ |
| recording_seconds | INTEGER | 録音時間（秒） |
| started_at | TIMESTAMP | 開始日時 |
| completed_at | TIMESTAMP | 完了日時 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### candle_presets（デフォルト画像管理）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | VARCHAR(100) | プリセット名 |
| on_image_url | TEXT | 点灯画像URL |
| off_image_url | TEXT | 消灯画像URL |
| is_default | BOOLEAN | デフォルトフラグ |

## 5. 非機能要件

### 5.1 パフォーマンス
- Whisper処理はバックグラウンド（非同期）
- 画像は適切にリサイズ・圧縮

### 5.2 セキュリティ
- パスワードはハッシュ化保存
- SQLインジェクション対策
- XSS対策

### 5.3 データ保持
- 音声ファイル: 文字起こし完了後削除
- 文字起こしテキスト: 永続保存
- 画像: S3互換ストレージに保存

## 8. 技術詳細メモ

### 8.1 UIライブラリ候補（iPad最適化）
| ライブラリ | 特徴 |
|-----------|------|
| **MUI (Material-UI)** | 豊富なコンポーネント、タッチ対応 |
| **Chakra UI** | シンプル、カスタマイズ容易 |
| **Mantine** | モダン、アニメーション豊富 |
| **Framer Motion** | アニメーション特化（併用推奨） |

推奨: **Mantine + Framer Motion**（ビジュアル重視のため）

### 8.2 Whisper統合
- `openai-whisper` または `faster-whisper`
- モデルサイズ: `small` または `medium`（精度/速度バランス）
- 日本語対応: `--language ja`
